# **SVG/PNG Export Plan — NDH/TISD Tensor‑Calculus Architecture**

This plan defines **how** diagrams will be exported, **which formats** are used, **how accessibility is preserved**, and **how enterprise caps are respected**.

It does **not** generate SVG/PNGs — it establishes the **rules and structure** for doing so.

---

## **1. Export Philosophy (Accessibility‑First)**

The NDH/TISD architecture requires:

- ASCII = **canonical layer**  
- SVG = **primary visual export**  
- PNG = **fallback for low‑bandwidth**  
- Alt‑text = **poverty‑grade accessibility**  
- No diagram may rely solely on graphical form  
- Every SVG/PNG must have a text‑native equivalent

This ensures:

- low‑income users  
- low‑bandwidth users  
- older devices  
- cognitive‑load‑sensitive users  

can access all content.

## **3. Export Constraints (Enterprise‑Cap Safe)**

To avoid hitting enterprise caps:

- SVGs must be **vector‑simple**  
- PNGs must be **low‑resolution**  
- No diagram may exceed **200 KB**  
- No export may contain **complex gradients**  
- No export may contain **embedded fonts**  
- No export may contain **multi‑layer masks**  
- No export may require **high‑density rendering**

This ensures:

- predictable file sizes  
- predictable rendering  
- predictable governance attachment behavior
